# AMEND.APPEND_ONLY.20260224.001 — Append-only protocol (no overwrites)

**Type:** C  
**Scope:** GOV  
**Effective:** 2026-02-24  
**Status:** ACTIVE

## Targets (amended)
This protocol applies to **all** Factorio Pocket Codex repo content, including:
- `MANUAL/**`
- `CODEX/**`
- `PATCHLIB/**`
- `INDEX/**`
- changelogs / registries / standards

## New ruling
1) **Do not modify or overwrite existing files.**  
2) Any change, correction, refinement, or expansion must be done by creating a **new file** that explicitly references the prior ruling(s).  
3) The repo is **append-only** from this date forward.

## Amendment file naming
Use dated, unique amendment IDs:

- `AMEND/<AREA>/AMEND.APPEND_ONLY.20260224.001.md` (global governance rulings)
- `AMEND/<AREA>/AMEND.<TARGET_ID>.YYYYMMDD.###.md` (topic-specific amendments)

## Required header fields for every amendment
- **Targets:** (IDs/files amended)
- **Amendment type:** clarify / extend / override / deprecate
- **Reason**
- **Effective date**
- **See also** (links)

## How to resolve “current truth”
- The **latest applicable amendment** (by date + sequence) overrides earlier text.
- Never delete older rulings; mark them as superseded via amendment notes.

# ARCH.BUS.001 — Main bus doctrine (your rules)

**Type:** C  
**Scope:** OPS  
**Stage:** Early/Mid  
**Tags:** #architecture #bus  
**Applies to:** V / M

## Rules
- Keep bus lanes consistent; don’t “braid” unless you standardize it.
- Add new lanes in a predictable spot (top/bottom rule).
- Split off with labeled subfactories; avoid spaghetti merges back into the bus.

## Transition note
When the bus becomes painful, start a city-block rail grid and migrate by subsystem.
